# Continuous Delivery

Software engineering approach where code changes are automatically built, tested, and prepared for release to production. Every change that passes automated tests is deployable.

## Core Principles

- Deployable at any point in time
- Automated testing for all changes
- Automated deployment pipeline
- Configuration as code

## Connection to DORA Metrics

Continuous delivery directly impacts [[dora-metrics]]: higher deployment frequency, lower lead time for changes, and lower change failure rate through smaller, more frequent releases.
