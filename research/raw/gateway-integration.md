# Paperclip ↔ OpenClaw Gateway Integration

## Status: CONNECTED ✅

## Architecture
- Paperclip server: localhost:3100 (embedded PostgreSQL, API-only mode)
- OpenClaw gateway: localhost:18789 (ws://)
- Adapter: `openclaw_gateway` with device auth (v2 format)

## Key Fix: Device Auth Protocol Version Mismatch
Paperclip's openclaw_gateway adapter (v0.2.7) uses v3 device auth payload format.
OpenClaw gateway (v2026.2.15) only supports v1/v2 format.

**v2 format:** `v2|deviceId|clientId|clientMode|role|scopes|signedAt|token|nonce`
**v3 format:** `v3|deviceId|clientId|clientMode|role|scopes|signedAt|token|nonce|platform|deviceFamily`

Patched `/tmp/paperclip/packages/adapters/openclaw-gateway/src/server/execute.ts`:
- Changed `buildDeviceAuthPayloadV3` to emit v2 prefix and omit platform/deviceFamily fields.

### Why disableDeviceAuth doesn't work
When no device object is sent, the gateway strips ALL scopes to empty (`scopes = []`).
This means every method authorization check fails with "missing scope: operator.write".
Device auth is required to preserve scopes.
