# Research: GitHub Project Management

How to use GitHub's built-in tools — Issues, Projects, Labels, Milestones — to manage work for a two-person team (Ξ2T × Sivart).

## Structure

```
github-project-management/
├── README.md                   ← you are here
├── best-practices.md           ← general best practices for GitHub project management
├── issues-best-practices.md    ← how to write and manage great issues
└── our-workflow.md             ← how we should use these tools
```

## Key Insight

GitHub's project management tools were built for software teams but work beautifully for any structured work. For a two-person team, the key is **keeping it lightweight enough to maintain but structured enough to be useful.** Too much process kills velocity. Too little loses context.
