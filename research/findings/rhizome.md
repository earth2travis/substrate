# Rhizome (Deleuze & Guattari)

## Definition

The **rhizome** (from *A Thousand Plateaus*, 1980) is [[gilles-deleuze|Deleuze]] and Guattari's model for non-hierarchical, non-arborescent structures. Unlike trees (roots, trunk, branches, hierarchy), rhizomes connect any point to any other point. Grass, fungi, and the internet are rhizomatic.

## Principles

1. **Connection**: Any point can and must connect to any other
2. **Heterogeneity**: Links across different registers and domains
3. **Multiplicity**: No unity, only multiplicities with changing dimensions
4. **Asignifying rupture**: A rhizome can be broken but will restart along old or new lines
5. **Cartography**: Rhizomes map rather than reproduce (no deep structure to copy)

## Connection to AI

The LLM Wiki pattern and knowledge graphs are rhizomatic — no single root or hierarchy, cross-references form from any concept to any other. This contrasts with traditional taxonomies and folder-based knowledge organization. Multi-agent systems operate rhizomatically: agents form ad-hoc connections rather than following rigid hierarchies.

## Connection to GitHub Knowledge Graph

The rhizome concept maps onto the [[github-as-memory]] "living project graph": GitHub issues, PRs, and commits form a rhizomatic structure where there's no central root — every node connects to many others. The graph emerges from work, not top-down design, just as rhizomatic growth happens through lateral connections.

Obsidian's backlink system [[obsidian]] creates rhizomatic knowledge structures — every wikilink is a new root emerging laterally.
