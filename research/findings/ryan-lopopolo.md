# Ryan Lopopolo

> OpenAI Frontier team member, harness engineering pioneer, author of the "Dark Factory" methodology where 0 lines of code were written by humans for a 1M+ line codebase.

## Key Facts

- **Role:** Member of the Technical Staff, OpenAI Frontier
- **Background:** Snowflake, Brick, Stripe, Citadel
- **Location:** OpenAI Bellevue, Washington (new office)
- **Notable:** Authored "[Harness engineering: leveraging Codex in an agent-first world](https://openai.com/index/harness-engineering/)" (Feb 11, 2026) — a viral article that defined the discourse in the field
- **First podcast:** His first podcast appearance was discussing these topics

## Work

- Built OpenAI's internal Symphony orchestrator
- Developed dark factory approach: no human-written code, all agent-generated
- Team grew from 3 to 7 engineers, maintaining increasing throughput as team scaled
- Authored official OpenAI article on harness engineering (Feb 2026)
- Uses Codex/Codex CLI extensively with a 1-week horizon to complete tasks under a minute

## Philosophy

- "Don't put agents in a box" -- give agents full accessibility over their domain
- Human review should be post-merge, not pre-merge (humans become bottleneck otherwise)
- Code as context: docs, tests, and review agents are ways to inject non-functional requirements into the system
- Everything is prompts: what the agent can't see is garbage
